## 🚀 主要更新

- 无

## 💡 功能优化

- 移除 公平抽取设置中的“最佳配置推荐/智能分析”入口及相关逻辑，避免误导用户调整无效或不适用的配置
- 优化 联动相关时间变化检测逻辑，改为按课程时间边界触发检查，降低后台轮询占用，并提升课前提前解禁、课前重置等场景的响应及时性
- 优化 自动更新分发链路，改为优先使用 SECTL 软件分发接口进行版本检测与下载，并保留 SECTL GitHub、GitHub 及镜像源回退，减少客户端直接访问 GitHub 与多源探测带来的流量消耗
- 优化 更新包安装流程，Inno 安装包使用标准静默参数执行，ZIP 更新包支持通过外部更新脚本在程序退出后解压覆盖并自动重启
- 优化 自动更新版本策略，自动检查、下载和安装仅允许在当前主版本内升级，避免 v2 自动跨版本更新到 v3
- 移除 更新页面中的“更新源选择”设置项，更新源现由 SECTL 分发接口与客户端自动回退链路统一管理

## 🐛 修复问题

- 修复 公平抽取权重未正确参与最终抽取的问题，确保频率、分组、性别、时间等权重按预期生效
- 修复 抽取后屏蔽仅降低权重、仍可能重复抽中的问题；屏蔽期内会优先从候选池排除，仅在可抽人数不足时降级补回
- 修复 公平抽取历史统计按 ID 匹配导致抽取次数读取不到的问题，改为与历史记录一致的姓名匹配
- 修复 平均值差值保护可能把高抽取次数对象纳入保护池的问题，现在优先保护低抽取次数对象并按差距阈值逐级扩展
- 修复 闪抽和抽奖分配学生时可能读取错误抽取方式配置的问题，确保点名、闪抽及名单独立配置互不串用
- 修复 打包版本将名单、音频、配置等运行数据误写入系统用户数据目录的问题，数据将继续保存在程序目录中
- 修复 关闭“浮窗启动时显示”后下次启动仍自动显示浮窗的问题
- 修复 启动软件时深色浮窗主题下文字颜色显示为黑色的问题
- 修复 自动更新模式下载完成后未继续自动安装的问题，现在“自动检查更新并安装”模式会在下载完成后直接启动安装流程
- 修复 更新包文件名仍按旧模板推导导致已下载文件无法识别或校验失败的问题，手动更新与自动更新现统一使用 SECTL 分发信息中的实际文件名
- 修复 SECTL 分发包定位错误导致下载接口 404 的问题，现在会使用项目平台 ID 获取 SecRandom 分发包，并优先通过 packageId 下载对应安装包
